# 미션 - 점심 메뉴 추천

## 구현 순서

### 1단계 - 코치, 음식 관련 핵심 도메인 설계
- [x] 코치들 도메인 설계
    - 코치 리스트를 가지고 있는 일급 컬렉션
    - 도메인 유효성 검사
        - 2~5명
- [x] 코치 도메인 설계
    - 값 : 이름, 못먹는 메뉴들, 선택된 메뉴들(월~금 순서[Queue]), 카테고리 카운트(HashMap 사용)
    - 도메인 유효성 검사
        - 2~4글자의 이름
- [x] 못 먹는 메뉴들
    - 값 : 메뉴 리스트
        - 메뉴는 0~2개
        - 존재하는 메뉴만 가능(각각의 메뉴를 전체 메뉴 도메인에 순회하여 검증)
        - 동일한 메뉴는 불가능
- [x] 전체 메뉴
    - 값 : 메뉴 리스트
- [x] 메뉴 카테고리 판별기(enum)
    - 순회해서 메뉴가 어떤 카테고리인지 판별하는 용도
- [x] 예외 메시지
  - 예외 메시지가 도메인과 테스트에 정합성을 지킬 수 있도록 하는 용도

### 2단계 - 메뉴 추천을 받을 코치 이름 입력
- [x] 메뉴 추천을 받을 코치의 이름 입력 요청 메시지 출력
- [x] [사용자] 코치 이름 입력
    - 입력 유효성 검사(구분자 검증)
- [x] 코치, 코치들 객체 생성
    - 도메인 유효성 검사(1단계에서 이미 개발)

### 3단계 - 못 먹는 메뉴 입력
- [x] 각 코치가 못 먹는 메뉴를 입력 요청 메시지 출력
- [x] [사용자] 못 먹는 메뉴 입력
  - 입력 유효성 검사(구분자 검증-2단계와 동일)
- [x] 못 먹는 음식들 객체 생성
- [x] 코치에 못 먹는 음식들 객체 넣기


### 4단계 - 월~금까지 추천 메뉴 랜덤 등록 및 추천 메뉴 출력
- [x] 테스트를 위해 랜덤한 메뉴를 선택하는 클래스를 인터페이스를 상속하도록 구성
- [x] 전체 메뉴에서 랜덤한 메뉴 선택
- [x] 선택한 메뉴 유효성 검사
  - 카테고리가 2개를 초과할 경우
  - 이미 선택된 메뉴일 경우
  - 못 먹는 음식인 경우
- [x] 코치에 선택한 메뉴 등록(월요일부터 금요일까지 반복)
- [x] 각 코치 별 월요일에서 금요일의 추천 메뉴 출력
